1. C++程序各部分说明

  • 注释： //

  • #include ：预处理器编译指令，用于包含头文件

    • #include <iostream>
// 该行预处理器编译指令等同于：将iostream文件的内容取代程序中的代码行#include <iostream>。原文件并没有被修改，而是将源代码文件与iostream文件组合成一个复合文件

    • 程序编译时自动执行

    • 头文件：每个头文件支持一组工具，被包含在其他文件中，也可以称为包含文件

      • 以.h结尾：C++与C的旧式风格，C与C++均可以使用
      • 没有扩展名：C++可以使用
      • 以c开头：转换后的C，可以使用一些不是C的特性，如namespace

  • int main( )：函数头

    • 接口：这里可以理解为操作系统与之的接口

  • using namespace：编译指令

    • using：编译指令

    • namespace

1.1 main函数

函数 = 函数头 + 函数体

1.1.1 函数头

函数头 = 接口

  • int：函数返回类型。描述的是从函数返回给调用它的函数的信息。
  • main：函数名
  • ()：这里面可以填写参数，称作形参列表或参数列表（argument list或parameter list）

函数头的书写注意事项：

  • 指定函数返回类型（如int，此后函数体内需要包含return 0语句），可以使用void类型，但是有些系统上不能工作，最好不用。
  • 传入的参数可以指定为void，表示无需传入参数

1.1.2 函数体

  • 语句以分号结束，不可省略
  • 结束函数

1.1.3 main()函数要求

  • 必须有main()函数，动态链接库（DLL）除外。
  • 如果编译器到达main()函数末尾没有返回语句，默认以return 0 语句结尾。但是这种隐含的返回语句只适用于main()函数

2.2 函数

  1. C++的函数分为两种：有返回值的和没有返回值的。

  2. 几个概念：

  • 调用函数

    • 最外层的接口函数

  • 函数调用

  • 被调用的函数

  1. C++程序应当为程序中使用的每个函数提供原型。函数原型之于函数就像变量声明之于变量，即指出涉及的类型，在使用某个函数前，必须提供原型。

  • 函数原型怎么提供？

    • 方法一：在源代码文件中输入函数原型。
    • 方法二：包含定义了原型的头文件。

  • 原型只描述了函数的接口，定义的时发送给函数的信息和返回的信息，不包含函数实现的代码。

  • 推荐写法：

    • 在开头提供自定义函数原型
