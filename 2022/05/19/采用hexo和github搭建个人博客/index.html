使用 Hexo+GitHub 搭建个人免费博客教程（小白向） - 知乎 (zhihu.com)

1. Hexo安装

  • 安装Node.js与git

    • 前往官网下载即可，无脑一直continue

    • cmd打开命令工具，查看安装是否成功

  • 安装Hexo

    • 执行安装命令： npm install -g hexo-cli

    • 进入自己想要书写博客的文件夹，此处建立在F盘下的“myblog”文件夹下

      执行完成访问http://localhost:4000，可以看到自己的博客

2. Github配置

  • 使用git连接github

    • 在git bash中设置用户名和邮箱，设置完成后会在.ssh下出现.gitconfig文件

  • 新建github仓库

    • 仓库名称为：用户名.github.io
    • 设置为public

  • 配置hexo的配置文件

    • 编辑“myblog/_config.yml”文件，修改文件末尾的deploy为如下设置。

    • 这里采用http连接，不采用ssh连接，因此无需配置ssh。repo后的连接就是http方式，这里填写自己的仓库链接就好。

    • branch网络上都给出的是master，但是这里自己实践如果推送到master会显示不了，仓库默认是main，因此这里把branch改成了main，后续也将会把内容都推送到main中。之后就可以正常显示了。

3. hexo主题配置

3.1 hexo文件夹说明

  • 初始化后的部分文件说明
    • _config.yml：网站的配置信息
    • package.json：应用程序的信息。比如一些依赖项的版本
    • scaffolds ：模板文件夹
    • source：资源文件夹。
      • 用于存放用户资源，除了_posts文件夹之外，开头命名为 _ 的文件夹或文件以及隐藏文件 将被忽略。
      • Markdown和HTML文件将被解析并放到public文件夹，其他文件会被拷贝过去。
    • themes：主题文件夹
      • Hexo会根据主题来生成静态页面

:star:一些杂记

_config.yml

  1. _config.yml使用的是yaml语法，同时也支持json格式，但是json格式相较于yaml更为复杂。
  2. 在模板中如“- - -”的符号改为“; ; ;”就能支持json

layout

  1. 默认有三种布局：post, draft, page。其中draft不会被显示在网页上。

  2. 创建新文章的命令为：

    hexo new post “new blog”，这里创建的就是post布局

    如果创建了草稿，并且需要迁移到post中，则需要：

    hexo new draft “new blog”

    hexo publish draft “new blog”

Scaffold

  1. 默认布局是三种，可以自己新建布局，存放scaffolds文件夹中。
  2. 默认的scaffolds中就存放了draft, page, post三个布局的md文件。

Front-matter

  1. categories参数：分类具有顺序性和层次性
  2. tags参数：标签没有顺序和层次
  3. comment参数：是否开启评论

永久链接

  1. 如果需要考虑使用恒定id，确保评论与文章不会映射混乱，可以使用插件hexo-abbrlink。
